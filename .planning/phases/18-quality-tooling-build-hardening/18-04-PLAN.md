<objective>
Audit silently-swallowed exception sites in the highest-value modules (cache, scanner, CLI supervisor) and annotate each site as either logged or intentional (SC5/QUAL-04).

Target scope: ~30-50 sites in the four modules that most directly affect diagnosability (cache, scanner, supervisor/cli). Remaining sites receive `// TODO-AUDIT: review exception handling` markers and are documented in the tracking note.

Purpose: The 183 silent catch blocks identified in SC5 are the root-cause obscurity behind issues like #71 (CLI timeout diagnosed only after adding logging). Each audited site either surfaces errors to the user via the project's logging facility, or carries an explicit rationale for why silence is correct.

Privacy constraint (NON-NEGOTIABLE): Log messages MUST NOT interpolate request bodies, API keys, bearer tokens, or any user-supplied secret. Only `e.message` and operational context (module name, operation name) are safe. This is the project's core privacy guarantee.

Output: Annotated source files (in-place edits to catch blocks) + exception-audit.md tracking note.
</objective>
