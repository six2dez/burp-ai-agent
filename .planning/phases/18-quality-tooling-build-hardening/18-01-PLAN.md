Fix the generateBuildFlags Gradle wiring (SC1/QUAL-05) and integrate detekt 1.23.8 as a blocking static-analysis gate with a committed baseline (SC2/QUAL-02-detekt).

Output: Updated build.gradle.kts, new detekt.yml, committed detekt-baseline.xml, updated build.yml.
</objective>

This passes the task's own `outputDir: DirectoryProperty` through the `TaskProvider`, so Gradle registers `generateBuildFlags` as the structural producer of that directory and infers the dependency for any consuming task — including `runKtlintCheckOverMainSourceSet` — without name-matching.
    3. Remove lines 111-113 entirely: the `tasks.matching { it.name.startsWith("runKtlint") }.configureEach { dependsOn(generateBuildFlags) }` block is superseded by the structural wiring. Leave the `tasks.withType<KotlinCompile> { dependsOn(generateBuildFlags) }` block at lines 101-107 untouched.
    4. Keep `val generatedSrcDir = ...` declaration even if unused (or remove it — it is only needed for the now-deleted srcDir call; removing it is cleaner).

    Fallback (if flatMap form does not infer the dependency after verification in Task 2): replace the new `kotlin.srcDir(...)` call with `kotlin.srcDir(project.files(generateBuildFlags.flatMap { it.outputDir }).builtBy(generateBuildFlags))` — per Assumption A2 in RESEARCH.md.

    SC2 — detekt plugin and config block in build.gradle.kts:

    5. Add `id("io.gitlab.arturbosch.detekt") version "1.23.8"` to the `plugins {}` block (after the existing cyclonedx line). Do NOT add `io.gitlab.arturbosch.detekt:detekt-formatting` as a detekt plugin dependency — it would double-gate ktlint style rules.
    6. Add a `detekt {}` configuration block after the `ktlint {}` block (after line 182):
       ```
       detekt {
           buildUponDefaultConfig = true
           allRules = false
           baseline = file("detekt-baseline.xml")
           parallel = true
           config.setFrom(files("detekt.yml"))
       }
       ```
       detekt 1.23.8 auto-wires its `detekt` task as a dependency of Gradle's `check` task — no explicit `tasks.check.dependsOn(tasks.detekt)` is needed.
