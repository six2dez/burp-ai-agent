<objective>
Lay the compile-time build gate and catalog foundation that every subsequent wave depends on. This plan (a) adds the -PstoreBuild Gradle property with a hand-rolled generateBuildFlags task that writes BuildFlags.kt into build/generated/buildflags, (b) renames the full-build artifact to Custom-AI-Agent-full-<version>.jar and keeps the store artifact as Custom-AI-Agent-<version>.jar, (c) adds nativeTool: Boolean = false to McpToolDescriptor, marks status and issue_create as nativeTool=true, and adds McpToolCatalog.available(storeBuild: Boolean = BuildFlags.STORE_BUILD) with the testable default-arg overload, and (d) writes three Wave-0 test stubs (red) that subsequent waves will turn green.

Purpose: Without BuildFlags.STORE_BUILD being a compile-time const, the store JAR cannot reliably exclude generic tools. Laying this first means all subsequent waves can reference BuildFlags.STORE_BUILD and McpToolCatalog.available() directly.

Output: Modified build.gradle.kts, modified McpToolCatalog.kt, three new test stub files.
</objective>
