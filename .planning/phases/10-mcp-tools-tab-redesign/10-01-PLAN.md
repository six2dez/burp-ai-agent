  <behavior>
    - groupTools(tools: List<McpToolDescriptor>): ToolGrouping — splits into native (nativeTool=true, sorted by title) and generic (nativeTool=false, sorted by title)
    - badgeStyle(tool: McpToolDescriptor): BadgeStyle — NATIVE when nativeTool=true; FULL when false (UI-04)
    - filterPredicate(query: String, tool: McpToolDescriptor): Boolean — true when query is blank OR tool.title.contains(query, ignoreCase=true) OR tool.description.contains(query, ignoreCase=true)
    - bulkToggleTargets(tools: List<McpToolDescriptor>, query: String, disabledIds: Set<String>): List<McpToolDescriptor> — returns tools where filterPredicate(query, tool) AND tool.id NOT IN disabledIds
    - ToolGrouping data class: native: List<McpToolDescriptor>, generic: List<McpToolDescriptor>
    - ToolRow not needed as a separate type — model works directly with McpToolDescriptor
    - categoryGroups(tools: List<McpToolDescriptor>): Map<String, List<McpToolDescriptor>> — groups generic tools by category, sorted by category name, tools in each group sorted by title (UI-03 Montoya sub-grouping)
  </behavior>

  <action>
    Create `src/main/kotlin/com/six2dez/burp/aiagent/ui/McpToolTabModel.kt` in package
    `com.six2dez.burp.aiagent.ui`.

    Import only: `com.six2dez.burp.aiagent.mcp.McpToolDescriptor` and
    `com.six2dez.burp.aiagent.ui.design.BadgeStyle`. NO javax.swing, java.awt, or JPanel imports.

    Declare `data class ToolGrouping(val native: List<McpToolDescriptor>, val generic: List<McpToolDescriptor>)`.

    Declare `object McpToolTabModel` with these four companion-style functions (top-level on the object):

    `fun groupTools(tools: List<McpToolDescriptor>): ToolGrouping` — partitions by `nativeTool`,
    each partition sorted ascending by `title` (case-sensitive String natural order). The native
    list contains only tools where nativeTool=true; the generic list contains tools where nativeTool=false.

    `fun badgeStyle(tool: McpToolDescriptor): BadgeStyle` — returns `BadgeStyle.NATIVE` if
    `tool.nativeTool`, else `BadgeStyle.FULL`. This is the sole source of truth for UI-04 badge
    assignment; the Swing layer in Plan 02 calls this and passes the result to `toolBadge()`.

    `fun filterPredicate(query: String, tool: McpToolDescriptor): Boolean` — returns true when
    query is blank (empty or whitespace-only) OR `tool.title.contains(query.trim(), ignoreCase = true)`
    OR `tool.description.contains(query.trim(), ignoreCase = true)`. Matching is on trimmed query;
    if query.trim() is empty the function short-circuits to true immediately.

    `fun bulkToggleTargets(tools: List<McpToolDescriptor>, query: String, disabledIds: Set<String>): List<McpToolDescriptor>` —
    returns the sublist of `tools` where `filterPredicate(query, tool)` is true AND `tool.id` is
    NOT in `disabledIds`. `disabledIds` represents the set of tool IDs whose checkboxes are
    `isEnabled = false` (pro-only or unsafe-locked tools); the Swing layer builds this set and passes it in.
    The function must NOT depend on Swing — it takes and returns plain `McpToolDescriptor` lists.

    `fun categoryGroups(tools: List<McpToolDescriptor>): Map<String, List<McpToolDescriptor>>` —
    groups the given tools by `category`, then sorts the resulting map by key (category name,
    case-sensitive natural order). Within each group, tools are sorted ascending by `title`.
    Returns a `LinkedHashMap` preserving insertion order (so iteration = alphabetical categories).
    This drives the Montoya section's category sub-header rendering.

    Do not add any logic beyond these five functions. No mutable state. No Swing dependency.
  </action>
