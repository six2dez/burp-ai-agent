<objective>
Migrate PassiveScanConfigPanel, ActiveScanConfigPanel, and ActiveScanQueuePanel to the Phase 9
design system, and add collapsible sections to the long scanner tabs (UI-06).

Purpose: Scanner panels are the most field-dense tabs. PassiveScanConfigPanel has ~25 constructor
fields; grouping them into collapsible sections (rate-limiting, cache, context caps, status) is the
primary UI-06 deliverable. ActiveScanConfigPanel is shorter but still has ad-hoc UiTheme usage and
raw button styling. ActiveScanQueuePanel (dialog) needs font/color token alignment.

These three files share no overlap with plan 01 or 03, so they run in Wave 1 in parallel.

NOTE: SettingsPanel.kt constructor call-site cleanup for these panels is handled in plan 11-03
(the single owner of SettingsPanel.kt). This plan only edits the panel files themselves.

Output: Both scanner config panels and the queue dialog use DesignTokens throughout; the passive
scanner panel groups its sections in AccordionPanel blocks for scannability.
</objective>
