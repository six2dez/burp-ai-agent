# Phase 9 â€” UI Design Contract: Design System Foundation

> Visual and interaction contract for Phase 9 (Design System Foundation) and the two phases that
> consume it: Phase 10 (MCP Tools Tab Redesign) and Phase 11 (Settings Tabs + Theme Rollout).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Context

**Tech stack:** Kotlin + Swing (JVM 21). No web framework. No shadcn / Tailwind / CSS.
**Platform constraint (ADR-2, locked):** Plain Swing + `javax.swing.UIManager` only. No JavaFX,
no FlatLaf, no third-party L&F libraries. The design system is a Kotlin module, not a CSS file.
**Light/dark:** Burp Suite changes its theme by calling `UIManager.setLookAndFeel()`. Tokens MUST
resolve dynamically from UIManager keys at call time (computed `get` properties), never cached.
**Existing seed:** `UiTheme.kt` already exists with `Colors`, `Typography`, and `isDarkTheme`.
This phase consolidates and extends it; it does NOT replace it from scratch.

## Spacing Scale

All values are multiples of 4. Exposed as `object DesignTokens.Spacing` (Int constants, pixels).

> **Note on `md` (12):** 12 is an intentional, multiple-of-4 intermediate step (labelâ†’field horizontal gaps). It is not in the abbreviated 4/8/16/24 reference set but is a standard spacing step in mainstream systems (Tailwind `spacing-3`, Material 12dp). Kept deliberately rather than forced to 8 or 16 â€” accepted exception to the abbreviated set.

| Token | Value | Usage |
|-------|-------|-------|
| `xs` | 4 | Icon-text gaps; accordion toggle padding; `addSpacerRow` minimum |
| `sm` | 8 | Row insets (vertical); button gap in button rows; section inter-spacing |
| `md` | 12 | Label-to-field insets inside form grid rows (horizontal) |
| `lg` | 16 | Tab content outer padding (horizontal); section header bottom gap |
| `xl` | 24 | Major section break between accordion panels |
| `sectionPad` | 8 | `sectionPanel` outer border (matches current `EmptyBorder(6,8,8,8)` â€” round to 8) |
| `formGridPad` | 8 | `formGrid` bottom border (rounds current `EmptyBorder(2,0,6,0)` up to the 4px grid) |

**Row inset constants (4-value Insets â€” top, left, bottom, right):**

| Token | Value | Usage |
|-------|-------|-------|
| `rowInsets` | Insets(4, 8, 4, 8) | Label cell in `addRowFull` / `addRowPair` (snapped from 3 â†’ 4px grid) |
| `fieldInsets` | Insets(4, 0, 4, 8) | Field cell in `addRowFull` (snapped from 3 â†’ 4px grid) |
| `fieldPairInsets` | Insets(4, 0, 4, 12) | Left field cell in `addRowPair` (snapped from 3 â†’ 4px grid) |
| `toggleRowInsets` | Insets(8, 0, 4, 0) | Toggle-switch field cell, extra top gap (snapped from 6 â†’ 8) |

Exceptions: `ToggleSwitch` fixed size is 44Ă—22 px (existing; keep as-is â€” not a spacing token).

## Typography

Font roles are computed `get` properties on `DesignTokens.Typography` (or `UiTheme.Typography`
after consolidation), deriving from `UIManager.getFont("Label.font")`. No hardcoded pt/px sizes;
all sizes are expressed as **multipliers** of the base font size so they scale with the user's OS
accessibility setting.

| Role | Derivation | Approximate px at base=13 | Weight | Usage |
|------|-----------|--------------------------|--------|-------|
| `sectionTitle` | `baseFont.deriveFont(Font.BOLD, base * 1.2f)` | ~16 | Bold (700) | Section header labels; accordion title |
| `body` | `baseFont.deriveFont(Font.PLAIN, base.toFloat())` | ~13 | Regular (400) | Row labels; checkbox text; description text; button labels |
| `caption` | `baseFont.deriveFont(Font.PLAIN, (base * 0.9f))` | ~12 | Regular (400) | Help/description labels under fields; badge text |
| `label` | `baseFont.deriveFont(Font.BOLD, base.toFloat())` | ~13 | Bold (700) | Inline labels that need visual weight (category headers in tool list; action buttons with emphasis) |
| `mono` | `UIManager.getFont("TextArea.font")` or `Font("Monospaced", PLAIN, base)` | ~13 | Regular (400) | JTextField/JTextArea with code/config content; password fields; URL fields |

**Line height:** Swing does not expose line-height as a property. Single-line labels: implicit.
Multi-line `JTextArea` / `JTextPane`: set `lineWrap = true`, `wrapStyleWord = true`; height is
determined by row count. No line-height constant is needed.

**Maximum declared roles: 5 (sectionTitle, body, caption, label, mono).** Phases 10 and 11 MUST
NOT introduce additional font derivations outside this set.

## Color

All color tokens are `get` properties resolving from UIManager at call time. Light/dark is handled
automatically because Burp sets UIManager when the user switches themes.

### Token Table

| Token | UIManager Key | Fallback (light) | Fallback (dark approx.) | Usage |
|-------|--------------|-----------------|------------------------|-------|
| `surface` | `Panel.background` | `#FFFBFF` | `#2B2B2B` | Panel and form backgrounds (60%) |
| `onSurface` | `Label.foreground` | `#1A1A1A` | `#EEEEEE` | Body text; row labels; checkbox text |
| `onSurfaceVariant` | `Label.disabledForeground` | `#666666` | `#AAAAAA` | Subtitle/description labels; section subtitles |
| `cardSurface` | `Table.background` or derive from `Panel.background` | `#F5F5F5` | `#333333` | Badge background; tool-row alternate background; card borders |
| `border` | `Component.borderColor` | `#CCCCCC` | `#555555` | Field borders; card outlines; 1px separators |
| `borderSubtle` | `Separator.foreground` | `#E0E0E0` | `#444444` | Accordion divider line; section dividers |
| `inputBackground` | `TextField.background` | `#FFFFFF` | `#3C3C3C` | JTextField; JTextArea; JPasswordField backgrounds |
| `inputForeground` | `TextField.foreground` | `#1A1A1A` | `#EEEEEE` | Input text |
| `primary` | `Burp.primaryButtonBackground` | `#D86633` | `#D86633` | Primary buttons; focus ring; toggle-on track |
| `onPrimary` | `Burp.primaryButtonForeground` | `#FFFFFF` | `#FFFFFF` | Text on primary-colored surfaces |
| `statusSuccess` | `Burp.successColor` | `#1B9E5A` | `#1B9E5A` | Running status; toggle-on state; store badge |
| `statusError` | `Burp.errorColor` | `#B3261E` | `#B3261E` | Error state; risk advisory accent |
| `statusWarning` | `Burp.warningColor` | `#F57C00` | `#F57C00` | Warning advisory accent |
| `badgeNative` | derived: `statusSuccess` at 15% opacity over `surface` | light: `#E8F5EE` | dark: see note | AI-native tool badge background |
| `badgeFull` | derived: `cardSurface` | light: `#F0F0F0` | dark: `#3A3A3A` | Full-build-only badge background |

**NOTE on `badgeNative`:** Swing does not support CSS opacity on solid colors. Approximate with
`Color(statusSuccess.red, statusSuccess.green, statusSuccess.blue, 40)` on a panel with
`isOpaque = false`, or use a 2-char alpha-blended solid (`Color(0xE8F5EE)` in light; detect via
`UiTheme.isDarkTheme` and use `Color(0x1E3A2C)` in dark).

### 60/30/10 Split

- **60% dominant:** `surface` â€” panel backgrounds, form backgrounds, scroll viewport
- **30% secondary:** `cardSurface` + `borderSubtle` â€” tool-row cards, accordion separators,
  badge backgrounds, input fields
- **10% accent (`primary`):** reserved for the following elements ONLY:
  - Primary action buttons (e.g., "Save Settings")
  - `ToggleSwitch` track when ON
  - Keyboard focus ring on interactive components
  - `statusSuccess` badge for "store" (native) indicator pill
  - Link-style secondary buttons (foreground only, transparent background)

**Destructive color (`statusError`):** reserved for:
  - `SubtleNotice.Level.RISK` advisory accent strip
  - "Delete" / "Reset" / irreversible confirmation buttons (border or text only, not fill)
  - Error state text (health-check failure, token validation failure)

## Component Builder API

Phase 9 consolidates existing ad-hoc builders from `SettingsPanel` into a public module at
`ui/design/`. The module MUST expose:

### 1. `DesignTokens` object

```
object DesignTokens {
    object Spacing { val xs=4; val sm=8; val md=12; val lg=16; val xl=24; val sectionPad=8; val formGridPad=8 }
    object Typography { /* get properties as described above */ }
    object Colors { /* get properties as described above â€” delegates to UiTheme.Colors or replaces it */ }
}
```

`UiTheme.kt` is either: (a) replaced by `DesignTokens` with the same properties, or (b) kept as
a thin shim delegating to `DesignTokens`. Both are acceptable. No new UIManager key lookups beyond
what already exists in `UiTheme.kt` without verifying the key exists in Burp's L&F.

### 2. `formGrid(): JPanel`

Returns a `JPanel(GridBagLayout)` with `background = Colors.surface` and
`border = EmptyBorder(0, 0, Spacing.formGridPad, 0)`. Client property `"row"` starts at 0.

Signature: `fun formGrid(): JPanel`

### 3. `addRowFull(grid: JPanel, labelText: String, field: JComponent, helpText: String? = null)`

Appends a label+field row to `grid`. Optional `helpText` adds a `caption`-styled `JLabel` on the
next grid row spanning columns 1-3 (below the field). Label is `body` font + `onSurface` color.
Small-component detection (JSpinner, JComboBox, JCheckBox, ToggleSwitch, JTextField â‰¤ 20 cols):
anchor WEST, fill NONE. Otherwise: fill HORIZONTAL.

Signature: `fun addRowFull(grid: JPanel, labelText: String, field: JComponent, helpText: String? = null)`

### 4. `addRowPair(grid: JPanel, leftLabel: String, leftField: JComponent, rightLabel: String, rightField: JComponent)`

Appends two label+field pairs on the same grid row (4 columns). Mirrors existing behavior.

Signature: `fun addRowPair(grid: JPanel, leftLabel: String, leftField: JComponent, rightLabel: String, rightField: JComponent)`

### 5. `addSpacerRow(grid: JPanel, height: Int = Spacing.xs)`

Inserts a rigid-area spacer row spanning all 4 columns. Default height = `Spacing.xs` (4 px).

Signature: `fun addSpacerRow(grid: JPanel, height: Int = DesignTokens.Spacing.xs)`

### 6. `sectionPanel(title: String, subtitle: String, content: JComponent): JPanel`

Returns a `BorderLayout` panel with a `BoxLayout(Y_AXIS)` header block (title in `sectionTitle`
font + `onSurface`; subtitle in `body` font + `onSurfaceVariant`; 4 px gap between them) and
content in `BorderLayout.CENTER`. Outer border: `EmptyBorder(Spacing.sectionPad)`.

Signature: `fun sectionPanel(title: String, subtitle: String, content: JComponent): JPanel`

### 7. `helpLabel(text: String): JLabel`

Returns a `JLabel(text)` styled with `caption` font and `onSurfaceVariant` foreground.
For multi-line help, callers should use `<html>` prefix in `text`.

Signature: `fun helpLabel(text: String): JLabel`

### 8. `primaryButton(label: String): JButton`

Returns a `JButton(label)` styled with `body` font, `primary` background, `onPrimary` foreground,
no border, `isFocusPainted = false`. Focus ring is drawn manually in `paintComponent` using a 2 px
`primary`-colored rounded rect when `isFocusOwner`.

**Alternative (simpler):** Set `isFocusPainted = true` and rely on L&F focus ring rather than
manual painting. Prefer this unless the L&F focus ring is invisible in dark mode.

Signature: `fun primaryButton(label: String): JButton`

### 9. `secondaryButton(label: String): JButton`

Returns a `JButton(label)` styled with `body` font, `surface` background, `primary` foreground,
`LineBorder(Colors.border, 1, true)`.

Signature: `fun secondaryButton(label: String): JButton`

### 10. `buildTabPanel(sections: List<JComponent>): JScrollPane`

Wraps sections in a vertical `BoxLayout(Y_AXIS)` panel inside a `JScrollPane` with no border and
`surface` viewport background. 8 px `Box.createRigidArea` gap between adjacent sections. Outer
padding: `EmptyBorder(Spacing.lg)` (16 px all sides).

Signature: `fun buildTabPanel(sections: List<JComponent>): JScrollPane`

### 11. `toolBadge(label: String, style: BadgeStyle): JLabel` (new â€” for Phase 10)

Returns a small `JLabel(label)` rendered as a pill badge. Badge specifications:

| BadgeStyle | Background | Foreground | Border | Text |
|------------|-----------|-----------|--------|------|
| `NATIVE` | `badgeNative` (green-tint) | `statusSuccess` | none | "store" |
| `FULL` | `badgeFull` (neutral) | `onSurfaceVariant` | none | "full" |

Properties: `caption` font; `EmptyBorder(2, 6, 2, 6)` insets; `isOpaque = true`; `corner =
6 px` â€” achieved by extending `JLabel` and overriding `paintComponent` with
`g2.fillRoundRect(0, 0, width, height, 6, 6)`.

Signature: `fun toolBadge(label: String, style: BadgeStyle): JLabel`
`enum class BadgeStyle { NATIVE, FULL }`

## Light/Dark Behavior Rules

Phases 10 and 11 MUST follow these rules without exception:

1. **No hardcoded Color literals in panel code.** All colors come from `DesignTokens.Colors`
   (computed `get` properties resolving from UIManager).
2. **Override `updateUI()` in every custom component** (JPanel subclass or component with manual
   painting) to re-apply theme colors after a L&F switch. Guard with an `initialized` flag or
   `lateinit` null-check to prevent NPE during super-constructor chain.
3. **`isDarkTheme` detection:** Use `UiTheme.isDarkTheme` (luminance of `surface` < 0.5).
   Permitted only in color tokens that have no suitable UIManager key (e.g., chat bubble colors,
   `badgeNative` dark variant). All standard surface/text/border colors must come from UIManager.
4. **`isOpaque` discipline:** Set `isOpaque = false` for overlaid panels (spacers, transparent
   wrappers). Set `isOpaque = true` for panels that own a background color.
5. **AccordionPanel**: already calls `updateUI()` correctly â€” keep as-is.
6. **SubtleNotice**: already calls `updateUI()` correctly â€” keep as-is.

## Usage Rules for Phases 10 and 11

**Phase 10 (McpConfigPanel redesign)** must:
- Use `sectionPanel` for the two top-level groups: "AI Tools (extension-native)" and
  "Montoya Tools (generic)".
- Use `toolBadge(BadgeStyle.NATIVE)` for tools where `nativeTool == true`; `toolBadge(BadgeStyle.FULL)` for generic tools.
- Use `helpLabel` for per-tool description text below the tool name in each row.
- Use `primaryButton` for "Enable all" / "Disable all" bulk toggles (or `secondaryButton` if bulk
  toggle is secondary to the master MCP toggle â€” executor decides based on visual hierarchy).
- Search/filter field: plain `JTextField` styled with `applyFieldStyle` (border = `Colors.border`,
  background = `Colors.inputBackground`, font = `Typography.body`). No custom painting.

**Phase 11 (Settings Tabs + Theme Rollout)** must:
- Replace ALL inline `JLabel(text).apply { font = ...; foreground = ... }` patterns in
  `SettingsPanel` and every panel under `ui/panels/` with calls to the design-system builders.
- Replace ALL inline `JPanel(GridBagLayout())` form patterns with `formGrid()` + `addRowFull` /
  `addRowPair` / `addSpacerRow`.
- Replace ALL inline `sectionPanel(...)` local private functions with the shared `sectionPanel`.
- `BackendConfigPanel.addRow()` / `addToggleRow()` / `addButtonRow()` may be refactored or left
  in place as private helpers that delegate to the public builders.
- `buildTabPanel()` in `SettingsPanel` must be replaced with the shared version.
- `AccordionPanel` title and subtitle font/color must resolve through `DesignTokens` (already does
  via `UiTheme` â€” remains compliant after consolidation).

**Both phases** must:
- Never introduce a `Color(0x......)` literal in panel code.
- Never introduce `Font(...)` constructors; use `DesignTokens.Typography` roles only.
- Never add spacing integers inline; use `DesignTokens.Spacing` constants.

## Flags (Non-Blocking)

The following items are noted for the executor's awareness. They do not block this contract.

- **FLAG-01:** `UiTheme.Colors.userBubble`, `.aiBubble`, `.userRole`, `.aiRole`, `.warningBannerBg`,
  `.warningBannerFg`, `.codeBlockBg`, `.inlineCodeBg` are chat-UI-specific tokens. They use
  `isDarkTheme` branching rather than UIManager keys because no Burp UIManager key maps to these
  roles. This is acceptable. Phase 9 should preserve them in `DesignTokens` or `UiTheme` as-is.
- **FLAG-02:** `cardSurface` token has no perfect UIManager key in all Burp L&F variants. Suggested
  fallback: derive as `Colors.surface.let { Color(it.red - 10, it.green - 10, it.blue - 10).coerceInRange() }` for light, or use `isDarkTheme` branch. Executor confirms the best approach.
- **FLAG-03:** `primaryButton` fill color (`Burp.primaryButtonBackground`) may render oddly in Burp
  Community's default L&F if the key is absent. The fallback `Color(0xD86633)` is already used by
  `UiTheme.Colors.primary` and is acceptable.
