  <action>
    Create `SsrfGuard` as an `object` in package `com.six2dez.burp.aiagent.util`.

    `fun isPrivateOrLinkLocal(url: String): Boolean`:
    1. If url is blank, return false.
    2. Parse the host from the URL: use `java.net.URI(url).host` inside a try-catch; if parsing throws (malformed), return false.
    3. If host is null or blank, return false.
    4. Parse the host to an InetAddress using `InetAddress.getByName(host)` inside a try-catch; if throws (UnknownHostException — no DNS needed for literal IPs, but possible for hostnames), return false. Do NOT perform DNS resolution for hostnames — only literal IP addresses should trigger the warning. Check this by calling `InetAddressUtils.isIPv4Address(host)` or verify `host` matches an IP pattern before calling getByName; alternatively call getByName only if the parsed host is a literal IP (numeric format).
    5. The classification logic (applied after getByName on a literal IP only):
       - `addr.isLoopbackAddress` → return false (loopback is excluded per D-01: "warn on loopback-excluded private")
       - `addr.isSiteLocalAddress` → return true (RFC-1918: 10.x, 172.16-31.x, 192.168.x)
       - `addr.isLinkLocalAddress` → return true (169.254.x.x and fe80::/10)
       - explicit check: `addr.hostAddress == "169.254.169.254"` — this is already covered by isLinkLocalAddress but add it explicitly for clarity and test visibility
       - otherwise → return false

    Implementation note on DNS: to avoid network calls, only classify literal IP addresses.
    Detect literal IP by trying `InetAddress.getByName(host)` and checking if the returned
    address `hostName == hostAddress` (no reverse DNS lookup was performed). Alternatively,
    check if `host` matches `\\d{1,3}(\\.\\d{1,3}){3}` (IPv4) or `[0-9a-fA-F:]+` (IPv6)
    before calling getByName. Using a regex guard is simpler and avoids the DNS edge case.
    If the host looks like a hostname (not a literal IP), return false immediately — the SSRF
    warning only fires on numeric IP addresses (hostname resolution would require DNS, which
    we do not call per D-01 "classify by address range inspection only").

    No imports beyond java.net.InetAddress and java.net.URI.
    No Montoya API, no OkHttp, no Swing.
  </action>
