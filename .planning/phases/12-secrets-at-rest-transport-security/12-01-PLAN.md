    Create `SecretCipher` as an `object` in package `com.six2dez.burp.aiagent.config`.
    Constructor takes `prefs: Preferences` (Burp Montoya Preferences interface — same import as in AgentSettings.kt).

    Master-key bootstrap (per D-01 — SEC-01):
    - Constant `MASTER_KEY_PREF_KEY = "secret.master.key.v1"` (keep this name in the source so plan 02 can reference it).
    - On first use, call `prefs.getString(MASTER_KEY_PREF_KEY)`. If null or blank: generate a 32-byte key via `SecureRandom().nextBytes(32)`, Base64-encode it, store with `prefs.setString(MASTER_KEY_PREF_KEY, encoded)`.
    - Decode the stored Base64 string to obtain the raw 32-byte key. Wrap it in `SecretKeySpec(bytes, "AES")`.
    - PBKDF2WithHmacSHA256 path: include a `fun deriveKey(passphrase: CharArray, salt: ByteArray): SecretKey` private fun using `SecretKeyFactory.getInstance("PBKDF2WithHmacSHA256")` at 600_000 iterations, 32-byte output. This path is NOT called by the current encrypt/decrypt but must be present for the future passphrase upgrade (per D-01).

    Encryption envelope (per STACK.md C2 / PITFALLS.md C2-1):
    - Generate a fresh 12-byte IV via `SecureRandom().nextBytes(12)` on every encrypt call.
    - Use `Cipher.getInstance("AES/GCM/NoPadding")` with `GCMParameterSpec(128, iv)`.
    - Ciphertext envelope stored in Preferences: `[1-byte version=0x01][12-byte IV][ciphertext+128-bit GCM tag]`, then Base64-encode the entire byte array.
    - Prefix the Base64 string with the literal prefix `"ENC1:"` so migration code can distinguish encrypted from plaintext values idempotently.

    `fun encrypt(plaintext: String, prefKeyName: String = ""): String`:
    - Returns `"ENC1:" + Base64.getEncoder().encodeToString(envelope)`.
    - `prefKeyName` is accepted but not used in the output; only used in catch blocks for safe logging.
    - If any exception is thrown during encryption, log `"SecretCipher.encrypt failed for key: $prefKeyName"` using `java.util.logging.Logger` (no Montoya dependency), then rethrow wrapped in `SecretCipherException`.

    `fun decrypt(ciphertext: String, prefKeyName: String = ""): String`:
    - If `ciphertext` does not start with `"ENC1:"`, return the value UNCHANGED (treat as plaintext — this is the migration compatibility path per D-01). This is correct behavior, not an error.
    - Strip the `"ENC1:"` prefix, Base64-decode, extract IV bytes [1..12] and ciphertext bytes [13..end] from the envelope (byte[0] is the version byte).
    - Decrypt with `Cipher.getInstance("AES/GCM/NoPadding")`. On `BadPaddingException` or `AEADBadTagException` (GCM tag authentication failure): log `"SecretCipher.decrypt failed for key: $prefKeyName — treating as empty"` (NEVER log the value), return `""` (fail-soft per D-01). Do NOT rethrow — the caller must never see an exception from decrypt().

    `class SecretCipherException(message: String, cause: Throwable) : RuntimeException(message, cause)` — ensure message does not include raw key material.

    Headless safety: the constructor only calls `prefs.getString`/`prefs.setString` and `SecureRandom`/`Base64` — no AWT/Swing imports. This satisfies the headless constraint (per D-01 / PITFALLS.md C2-3).

    No imports from `burp.api.montoya` other than `burp.api.montoya.persistence.Preferences`. No OkHttp, no external libraries.

    Write tests first (RED), then implement (GREEN). Use `InMemoryPrefs` mock pattern from AgentSettingsMigrationTest.kt.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Information Disclosure | SecretCipher catch blocks | mitigate | Log only the preference key name (String param), never the plaintext or ciphertext; enforced in action spec |
| T-12-02 | Tampering | MASTER_KEY_PREF_KEY Preferences entry | accept | Per-install key in Burp prefs raises bar over plaintext; any attacker who can write prefs can also read them; documented residual risk |
| T-12-03 | Spoofing | Rolling-own-crypto (bad cipher mode) | mitigate | Mandatory "AES/GCM/NoPadding" + GCMParameterSpec(128, iv) — authenticated encryption; IV is fresh per call via SecureRandom; PBKDF2 at 600k iterations |
| T-12-04 | Elevation of Privilege | Weak/hardcoded master key | mitigate | 32-byte SecureRandom key generated at first use; never derived from a hardcoded constant |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this plan — javax.crypto is JDK built-in |
</threat_model>
