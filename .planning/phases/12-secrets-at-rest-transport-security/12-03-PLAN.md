<objective>
Fix the argv password exposure in McpTls.generateSelfSigned() (SEC-02 / D-01 locked
decision A3) by keeping the existing keytool ProcessBuilder approach but passing the
password via the child process environment instead of the command-line argument list.

Purpose: Eliminate the -storepass/-keypass literal value on the argv (visible in `ps aux`
during keytool execution). The fix replaces `-storepass passStr` / `-keypass passStr` with
`-storepass:env KS_PASS` / `-keypass:env KS_PASS` and sets `pb.environment()["KS_PASS"] =
passStr` before `.start()`. No new runtime dependency. No --add-exports JVM flag. No
sun.security.* usage (which would compile fine but throw IllegalAccessError inside Burp's
JVM at runtime because Burp does not export sun.security.tools.keytool).

Output: Patched McpTls.generateSelfSigned() + McpTlsInJvmTest.kt asserting no literal
password on argv and that the generated PKCS12 file loads correctly.
</objective>
