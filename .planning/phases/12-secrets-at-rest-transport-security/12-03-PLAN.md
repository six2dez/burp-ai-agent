<objective>
Replace the keytool subprocess in McpTls.generateSelfSigned() with an in-JVM implementation
that uses JDK 21 classes to generate an RSA-2048 self-signed certificate and store it in a
PKCS12 KeyStore file.

Purpose: Eliminate the -storepass/-keypass argv exposure (SEC-02 / D-01 locked decision A3).
The fix removes the last subprocess that had the keystore password visible in `ps aux` during
keytool execution, and removes the dependency on keytool being present on PATH. No new
runtime dependency — javax.security, java.security, and sun.security.x509 are JDK 21 built-ins.

Output: Rewritten McpTls.generateSelfSigned() + McpTlsInJvmTest.kt verifying the in-JVM
cert generates, writes, and loads back correctly.
</objective>

    The recommended implementation for JDK 21 Burp-embedded JVM (Claude's discretion per D-01):
    Use `sun.security.x509.CertAndKeyGen`:
    ```
    val certAndKey = CertAndKeyGen("RSA", "SHA256withRSA")
    certAndKey.generate(2048)
    val validFrom = Date()
    val validity = 365L * 24 * 60 * 60  // 365 days in seconds
    val cert = certAndKey.getSelfCertificate(X500Name("CN=burp-mcp"), validFrom, validity)

    val ks = KeyStore.getInstance("PKCS12")

    keystoreFile.parentFile?.mkdirs()
    keystoreFile.outputStream().use { out -> ks.store(out, password) }
    ```
