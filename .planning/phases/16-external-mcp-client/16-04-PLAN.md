<objective>
Wire `ExternalMcpClientManager` into the agent's tool dispatch path: extend `McpToolContext` with the optional manager field, fan external tool descriptions into `describeTools()`, and route `ext:`-prefixed calls through `executeToolResult()` to the external server with redacted outbound arguments.

Purpose: This plan closes SC1 (external tools visible in preamble) and SC2 (trust-boundary wrap in tool results) from the tool-dispatch side. The trust-boundary marker was applied by Plan 03; Plan 04 ensures it flows untouched into the AI prompt. D-03 (outbound privacy) is enforced here by running `context.redactIfNeeded()` on the outbound args JSON before forwarding to `ExternalMcpClientManager.callTool()`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent AI context ← external tool result | Trust-boundary-wrapped result from ExternalMcpClientManager enters the AI prompt; wrapper ensures AI cannot confuse it with system instructions |
| McpTools outbound args → external server | Tool-call arguments redacted before leaving Burp toward third-party server (D-03: outbound privacy) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-04-PI | Tampering | External tool result in AI context | mitigate | Results from `callTool()` already carry `[EXTERNAL-TOOL-RESULT:...]` wrapper from Plan 03; advisory note added to tool preamble in `describeTools()` explicitly instructs AI to treat content as user-supplied data; acceptance criterion asserts advisory text present |
| T-16-04-PV | Information disclosure | Outbound tool-call arguments to external server | mitigate | `context.redactIfNeeded(argsJson)` applied before forwarding to `ExternalMcpClientManager.callTool()` per D-03 (locked user decision); acceptance criterion asserts `redactIfNeeded` present in `ext:` branch |
| T-16-04-COL | Spoofing | ext: tool name colliding with built-in | mitigate | `startsWith("ext:")` check is FIRST in `executeToolResult()` — built-ins only reached when name does NOT start with `ext:`; built-in catalog lookup only occurs after the early-return branch |
| T-16-04-NULL | Denial of service | Null externalClientManager when ext: tool called | accept | Returns `errorResult("External MCP client not available")` — graceful degradation; user is informed; no crash |
</threat_model>
