    Class `ExternalMcpClientManager(private val cipher: SecretCipher)`:

    Internal state per connected server stored as a private data class `ServerConnection` holding: `config: ExternalMcpServerConfig`, `client: Client?`, `transport: SsrfGuardClientTransport/Transport?` (note: use `io.modelcontextprotocol.kotlin.sdk.client.Transport` as the type), `process: Process?`, `scope: CoroutineScope`, `job: Job?`, `stateRef: AtomicReference<ExternalMcpConnectionState>`, `cachedTools: List<ExternalToolDescriptor>`.

    Key methods:

    `fun start(configs: List<ExternalMcpServerConfig>)`:
    - Create one `ServerConnection` per enabled config
    - For each SSE config: call `SsrfGuard.isPrivateOrLinkLocal(config.url)` as a check (log a warning if true ‚Äî the UI already showed a soft warning; here it is just logged, never blocked)
    - Create `HttpClient(CIO) { install(SSE) }` ‚Äî one per manager instance (not per server)
    - Launch `connectServer(serverName, config)` in the shared `CoroutineScope(Dispatchers.IO + SupervisorJob())`

    `private suspend fun connectServer(serverName: String, config: ExternalMcpServerConfig)`:
    - Set state to `Connecting`
    - Build transport based on `config.transport`:
      - SSE: `SseClientTransport(httpClient, config.url) { if (decryptedToken.isNotBlank()) headers.append("Authorization", "Bearer $decryptedToken") }` where `decryptedToken = cipher.decrypt(config.encryptedToken, "mcp.external.servers.v1")` ‚Äî decrypted transiently in the lambda; never stored in any field
      - STDIO (only if `config.transport == ExternalMcpTransport.STDIO`): use `ProcessBuilder(config.command + config.extraArgs)` ‚Äî the argument is a `List<String>`, NOT a String (prevents shell expansion per security requirement); set `redirectErrorStream(true)`, `redirectInput(ProcessBuilder.Redirect.PIPE)`, `redirectOutput(ProcessBuilder.Redirect.PIPE)`; inject only `config.envVars` into `pb.environment()` ‚Äî do NOT call `inheritIO()` or inherit parent env; store the `Process` in the connection's `process` field; wrap stdin/stdout with `StdioClientTransport(process.inputStream.asSource().buffered(), process.outputStream.asSink().buffered())`
    - Create `Client(Implementation("burp-ai-agent-ext", "0.9.0"), ClientOptions())`
    - Call `client.connect(transport)` (suspend)
    - Call `client.listTools(ListToolsRequest(), null)` and cache as `List<ExternalToolDescriptor>` with names prefixed `ext:<serverName>:<originalToolName>`
    - Set state to `Connected(toolCount)`

    `fun availableTools(): List<ExternalToolDescriptor>`:
    - Returns flattened list of all `cachedTools` across all connected servers
    - Thread-safe: reads from `@Volatile` field or `CopyOnWriteArrayList`

    `suspend fun callTool(serverName: String, toolName: String, args: Map<String, Any?>): String`:
    - Find connection by `serverName`
    - Call `client.callTool(toolName, args, false, null)` (note: `toolName` here is the raw name without prefix ‚Äî strip the `ext:<serverName>:` prefix before calling)
    - Extract text: `callResult?.content?.filterIsInstance<TextContent>()?.joinToString("\n") { it.text.orEmpty() } ?: ""`
    - Apply trust boundary: `wrapWithTrustBoundary(serverName, rawText)`
    - Emit audit: `AuditLogger.emitGlobal("external_mcp_call", buildMap { put("server", serverName); put("tool", toolName); put("status", "ok") })` ‚Äî keep entire `buildMap {}` inside the `if (AuditLogger.isEnabled())` gate (per Phase 18 CR-02 allocation guard)
    - On exception: emit audit with `"status" to "error"`, `"error" to e.message.orEmpty()` (never log token); return error string

    `private fun wrapWithTrustBoundary(serverName: String, rawResult: String): String`:
    - Returns `"$TRUST_BOUNDARY_OPEN$serverName]\n$rawResult\n$TRUST_BOUNDARY_CLOSE"`

    `fun stop()`:
    - For each connection: cancel job, then `runBlocking { withTimeoutOrNull(5000) { transport?.close() }; withTimeoutOrNull(5000) { client?.close() } }`; then `process?.destroyForcibly()` for stdio servers
    - Cancel the shared scope
    - Mirror exactly the `McpStdioBridge.stop()` pattern from `16-PATTERNS.md`

    `fun connectionState(serverName: String): ExternalMcpConnectionState`:
    - Returns current `stateRef.get()` for the named server; returns `Disconnected` if not found

    Reconnect logic: on connect failure, schedule a retry via `ScheduledExecutorService` (reuse `scheduler` from McpSupervisor pattern) with exponential backoff: `RECONNECT_DELAY_BASE_MS * (2.0.pow(attempt)).toLong()`, capped at 30_000ms; max `MAX_RECONNECT_ATTEMPTS` = 3; after exhausting retries set state to `Error(message)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ExternalMcpClientManager ‚Üí External MCP server (SSE) | Outbound ktor-client SSE connection carries bearer token; token must be decrypted transiently only in request lambda |
| ExternalMcpClientManager ‚Üí stdio subprocess | Process is spawned from user-configured command; subprocess can produce arbitrary stdout |
| External MCP server ‚Üí Agent AI context | Tool results (untrusted data) cross into the AI prompt ‚Äî must be wrapped in trust-boundary marker |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03-CMD | Elevation of privilege | stdio subprocess (`ProcessBuilder`) | mitigate | `ProcessBuilder(List<String>)` form only ‚Äî NO `Runtime.exec(String)`, no shell, `redirectErrorStream(true)`, no `inheritIO()`, only user-configured `envVars` injected; acceptance criterion asserts `ProcessBuilder(String)` form is absent |
| T-16-03-PI | Tampering | External tool result entering AI context | mitigate | `wrapWithTrustBoundary()` called in `callTool()` before any return ‚Äî result is ALWAYS wrapped; test `trustBoundaryWrap_addsCorrectMarkers` asserts markers present; system prompt advisory added in Plan 04 |
| T-16-03-TL | Information disclosure | Bearer token in SSE requestBuilder lambda | mitigate | Token decrypted transiently inside the lambda only (`cipher.decrypt()`); never assigned to a field or logged; exception path logs only key name (`"mcp.external.servers.v1"`) |
| T-16-03-ZOM | Denial of service | stdio process zombie on extension unload | mitigate | `stop()` calls `process?.destroyForcibly()` after `client.close()` and within a `runBlocking { withTimeoutOrNull(5000) {...} }` block; test `stop_destroysStdioProcess` asserts `destroyForcibly()` is called |
| T-16-03-COL | Spoofing | External tool name collision with built-in tools | mitigate | `availableTools()` prefixes ALL external tool names with `ext:<serverName>:` unconditionally; built-in tool dispatch in Plan 04 checks `ext:` prefix before routing |
</threat_model>
