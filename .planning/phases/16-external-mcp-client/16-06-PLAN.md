  <how-to-verify>
    **SC5: Fat JAR loads in live Burp (Path A ClassLoader smoke test)**

    1. Locate the built JAR: `ls build/libs/Custom-AI-Agent-*.jar`
    2. Open Burp Suite (Community or Professional)
    3. Extensions → Add → select the JAR
    4. Confirm: no `NoClassDefFoundError` or `ClassNotFoundException` in Burp's Output tab or Errors tab
    5. Confirm: "Burp AI Agent" appears in the Extensions list with no error icon
    6. Confirm: MCP Tools tab opens and is responsive

    **SC1: External MCP server connection (real server)**

    Choose either option A (SSE) or option B (stdio) — both are preferred; at minimum one must pass.

    *Option A — SSE server:*
    1. Start a local SSE MCP server (e.g., `npx -y @modelcontextprotocol/server-filesystem /tmp` in SSE mode, or any real external server)
    2. In Burp → AI Agent → Settings → MCP tab → scroll to "External MCP Servers" accordion
    3. Click "Add Server"
    4. Name: `test-sse`, Transport: SSE, URL: `http://127.0.0.1:<port>/sse`, token: blank
    5. Click "Save Server"
    6. Observe: server row appears in the list; Status column shows "Connected (N tools)" (green dot)
    7. Open the AI Agent chat; send a message asking the agent to list its tools
    8. Confirm: `ext:test-sse:<toolname>` entries appear in the agent's tool preamble response

    *Option B — stdio server:*
    1. Ensure `stdioEnabled` is enabled in MCP settings
    2. Click "Add Server", Transport: stdio
    3. Command: `npx -y @modelcontextprotocol/server-filesystem /tmp` (or another locally available MCP server)
    4. Click "Save Server"; confirm the stdio local-process SubtleNotice (RISK banner) was visible in the form
    5. Observe: Status column shows "Connected (N tools)"
    6. Confirm ext: tools appear in the agent tool preamble

    **SC3: SSRF warning (optional quick check)**
    1. Open "Add Server" form, transport SSE
    2. Type `http://192.168.1.100/sse` in the URL field
    3. Confirm: yellow SSRF warning label appears below the field immediately (while typing)

    **SC4: Token encryption (optional quick check)**
    1. Add a server with a bearer token (any text)
    2. Close and reopen Burp (or restart the extension)
    3. Open the server edit form — token field shows masked characters (JPasswordField)
    4. Confirm the settings load correctly and the server reconnects
