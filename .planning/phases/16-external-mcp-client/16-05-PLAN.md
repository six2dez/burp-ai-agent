<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI form → ExternalMcpServerConfig | User-typed fields (name, URL, command, token, env vars) cross into a config object; validation must happen before persistence |
| JPasswordField → bearerToken field | Bearer token raw text is held in JPasswordField in memory; it must NOT be logged, and encryption must happen at persistence time in AgentSettings.saveExternalMcpServers() (not in the UI panel) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-05-SSRF | Tampering | SSE URL field in ExternalServersPanel | mitigate | `SsrfGuard.isPrivateOrLinkLocal()` fires on every `DocumentEvent` on the URL field; SSRF warning label becomes visible immediately (non-blocking — user can proceed); acceptance criterion asserts `SsrfGuard.isPrivateOrLinkLocal` present in file |
| T-16-05-CMD | Elevation of privilege | Command field for stdio transport | mitigate | `SubtleNotice(Level.RISK)` always visible when transport=stdio; warns user explicitly that a local process will be spawned; command is stored as a `List<String>` (parsed from form), not a raw shell string |
| T-16-05-TL | Information disclosure | Bearer token in JPasswordField | mitigate | `JPasswordField` masks text by default (`echoChar='*'`); token value is never logged; bearerToken field in ExternalMcpServerConfig holds PLAINTEXT; encryption happens in `AgentSettings.saveExternalMcpServers()` at persistence time (Plan 02), not in UI; panel accepts plaintext from load (decrypted in AgentSettings) and passes plaintext to save (encrypted in AgentSettings) |
| T-16-05-VAL | Tampering | Invalid/malicious server name or URL | mitigate | Validation on Save Server: name required and no duplicate; URL must start with `http://` or `https://`; command required for stdio; `LineBorder(Colors.statusError)` and `helpLabel()` feedback on offending field |
</threat_model>
