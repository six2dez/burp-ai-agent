# Phase 16 — UI Design Contract: External MCP Client

> Visual and interaction contract for the external MCP server CRUD UI.
> Generated by gsd-ui-researcher; consumed by gsd-planner and gsd-executor.
>
> **Platform note:** This is a Swing UI inside a Burp Suite extension. No CSS, no HTML,
> no React, no Tailwind. All tokens reference `DesignTokens.kt`; all primitives reference
> `Components.kt`. The constraint is absolute.

## Spacing Scale

All values derive from `DesignTokens.Spacing`. No new tokens are introduced.

| Token | Pixel Value | Swing Usage |
|-------|-------------|-------------|
| `Spacing.xs` | 4 px | `addSpacerRow()` between form rows; icon/label gaps; `Box.createRigidArea(Dimension(4,0))` |
| `Spacing.sm` | 8 px | Gap between buttons in toolbar rows; `Box.createRigidArea(Dimension(8,0))` between Edit/Remove buttons; notice wrapper top margin |
| `Spacing.md` | 12 px | Label-to-field horizontal inset (via `fieldPairInsets`); not used directly in new code |
| `Spacing.lg` | 16 px | Outer tab content padding (inherited from `buildTabPanel`); section separator top gap |
| `Spacing.xl` | 24 px | Not used directly in this panel; inherited where `buildTabPanel` applies it |

## Transport Badge Specification

A new `BadgeStyle` variant is NOT added to the enum. Instead, the transport badge reuses the existing `toolBadge()` builder with a deliberate style mapping:

| Transport | `toolBadge()` style | Badge text | Background token | Text token |
|-----------|---------------------|------------|------------------|------------|
| SSE | `BadgeStyle.FULL` | `"SSE"` | `Colors.cardSurface` | `Colors.onSurfaceVariant` |
| stdio | `BadgeStyle.NATIVE` | `"stdio"` | `Colors.badgeNative` | `Colors.statusSuccess` |

**Rationale:** SSE is the normal/neutral case (FULL = neutral gray). stdio is a more sensitive capability (NATIVE = green-tinted) to create visual salience — the "this is different" affordance. No new badge styles are created.

**Source:** `Components.kt` `toolBadge()` + `BadgeStyle`; design principle from Phase 9 UI-SPEC.

## Component Layout: ExternalServersPanel

The external-server CRUD UI is delivered as a new `ExternalServersPanel` class, mounted inside the existing `McpConfigPanel` as a second `AccordionPanel` (collapsed by default, below the preprocessing accordion). Its layout follows the same stack pattern established in `McpConfigPanel.kt`.

### Panel Hierarchy

```
sectionPanel("External MCP Servers", "Connect to external/custom MCP server tools.", body)
  └── AccordionPanel(title="External MCP Servers", subtitle="...", initiallyExpanded=false)
        └── JPanel (BoxLayout Y_AXIS) — "stack"
              ├── [HEADER ROW] JPanel (FlowLayout LEFT) — server count label + "Add Server" button
              ├── [SERVER LIST] JScrollPane → JTable (ExternalServerTableModel)
              │     columns: [Enable] [Name] [Transport] [Status] [Actions]
              └── [ADD/EDIT FORM] AccordionPanel or card, collapsed by default
                    expanded when user clicks "Add Server" or "Edit" on a row
```

### 1. Server List (JTable)

| Property | Value |
|----------|-------|
| Component | `JTable` with `ExternalServerTableModel : AbstractTableModel` |
| Background | `DesignTokens.Colors.surface`; alternate rows use `DesignTokens.Colors.cardSurface` |
| Font | `DesignTokens.Typography.body` for cell text; `DesignTokens.Typography.label` for column headers |
| Row height | 24 px |
| Selection mode | `ListSelectionModel.SINGLE_SELECTION` |
| Auto resize | `JTable.AUTO_RESIZE_LAST_COLUMN` |

**Columns:**

| # | Label | Width | Content | Renderer notes |
|---|-------|-------|---------|----------------|
| 0 | Enable | 56 px | `JCheckBox` (enable/disable toggle) | `Boolean` column with checkbox renderer; label omitted |
| 1 | Name | 160 px (preferred) | Server display name string | `Typography.body`; `Colors.onSurface`; if disabled: `Colors.onSurfaceVariant` |
| 2 | Transport | 72 px | Transport badge — see badge spec above | Custom `TableCellRenderer` painting `toolBadge()` component |
| 3 | Status | 120 px (preferred) | Status dot + label — see status states below | Custom renderer; dot is a 8×8 oval painted via `Graphics2D.fillOval` |
| 4 | Actions | 108 px | "Edit" + "Remove" buttons inline | Custom `TableCellRenderer` + `TableCellEditor` with two `secondaryButton()` instances; 8 px gap between them |

**Status dot colors:**

| State | Dot color token | Label text | Label color token |
|-------|-----------------|------------|-------------------|
| Connected | `Colors.statusSuccess` | "Connected (N tools)" | `Colors.statusSuccess` |
| Connecting | `Colors.statusWarning` | "Connecting..." | `Colors.statusWarning` |
| Retrying | `Colors.statusWarning` | "Retrying (N/3)..." | `Colors.statusWarning` |
| Error | `Colors.statusError` | "Error" | `Colors.statusError` |
| Disabled | `Colors.onSurfaceVariant` | "Disabled" | `Colors.onSurfaceVariant` |

**Empty state (zero servers configured):**

Displayed as a single full-width row inside the JTable area. A `JLabel` overlaid or injected as a no-results row renderer:

- Text: `"No external MCP servers configured. Click 'Add Server' to register one."`
- Font: `Typography.body`
- Color: `Colors.onSurfaceVariant`
- Alignment: centered horizontally and vertically within the scroll pane viewport

**List header row** (above JTable, outside of it):

```
[JPanel FlowLayout.LEFT, 4px vgap, 8px hgap]
  JLabel("External Servers (N)")  — Typography.label, Colors.onSurface
  [rigid area 8px]
  primaryButton("Add Server")     — Colors.primary bg, Colors.onPrimary fg
```

When `stdioEnabled` is false (global MCP setting), the "Add Server" button remains enabled but the transport selector in the Add/Edit form defaults to SSE and hides the stdio option.

### 2. Add/Edit Server Form

The form is displayed:
- As an inline expansion below the table when "Add Server" is clicked (same `BoxLayout Y_AXIS` stack — a new JPanel `formCard` slides in via `isVisible = true` + `revalidate()`).
- The same form is reused for "Edit" — fields pre-populated from the selected server config.
- A "Cancel" button collapses the form without saving.

**Form title bar:**

```
JLabel("Add External Server") / JLabel("Edit: {name}")
  — Typography.sectionTitle, Colors.onSurface
  — EmptyBorder(Spacing.sm, 0, Spacing.sm, 0)
```

**Form fields** (all via `formGrid()` + `addRowFull()` / `addRowPair()`):

| Label | Field type | Component | Notes |
|-------|-----------|-----------|-------|
| Display name | `JTextField` | `applyFieldStyle()` | Required; validation error if blank |
| Transport | `JComboBox<String>(["SSE", "stdio"])` | Body font; default "SSE" | Switching transport shows/hides the SSE vs stdio sub-fields (see below); stdio option is hidden (removed from model) when `mcpSettings.stdioEnabled == false` |
| Enabled | `ToggleSwitch` | `Spacing.toggleRowInsets` | Defaults to `true` for new servers |

**SSE sub-fields** (visible when transport = SSE):

| Label | Field | Notes |
|-------|-------|-------|
| Server URL | `JTextField` (mono, `applyFieldStyle`) | SSRF inline warning fires on URL change (see SSRF warning spec); help text: "SSE endpoint URL (e.g. https://myserver.example.com/sse)" |
| Bearer token | `JPanel` (X_AXIS) containing `JPasswordField` + `JButton("Show")` | Show/hide toggle — see token field spec below; help text: "Optional; stored encrypted at rest" |

**stdio sub-fields** (visible when transport = stdio; hidden by default):

| Label | Field | Notes |
|-------|-------|-------|
| Command | `JTextField` (mono, `applyFieldStyle`) | Required; e.g. `npx -y @modelcontextprotocol/server-filesystem /tmp`; help text: "Full command to spawn the MCP server process" |
| Arguments (extra) | `JTextField` (mono, `applyFieldStyle`) | Space-separated; optional; pre-parsed into a list on save; help text: "Additional arguments (space-separated)" |
| Environment vars | `JTextArea` (3 rows, `applyAreaStyle`) | One `KEY=VALUE` per line; optional; help text: "KEY=VALUE pairs (one per line); only these vars are passed to the process" |

**stdio local-process warning** (always visible when transport = stdio):

```
SubtleNotice(Level.RISK).setMessage(
  level = Level.RISK,
  html = "<b>Warning:</b> This server will run a local process using the command above.
          Only configure commands you trust. The process is spawned with no shell expansion
          and only the environment variables you provide."
)
```

Placed immediately below the Environment vars row, above the button row. `isVisible = true` as soon as stdio transport is selected.

**Form buttons row** (`JPanel`, `BoxLayout.X_AXIS`):

```
primaryButton("Save Server")       — Colors.primary
Box.createRigidArea(Dimension(8,0))
secondaryButton("Cancel")          — Colors.surface bg, Colors.primary fg, border Colors.border
```

### 3. SSRF Inline Warning (SSE URL field)

Pattern: identical to `BackendConfigPanel.ssrfWarningLabel`.

```kotlin
// Instantiation
val ssrfWarning = JLabel(
    "Warning: this URL resolves to a private/internal address — verify this is intentional"
).apply {
    foreground = DesignTokens.Colors.statusWarning
    font = DesignTokens.Typography.caption
    isVisible = false
}
```

Trigger: on every URL field `DocumentEvent` (keystrokes), call `SsrfGuard.isPrivateOrLinkLocal(url)`. If true, `ssrfWarning.isVisible = true`. Non-blocking — user can proceed with a private URL.

Placement: a `helpLabel`-style row immediately below the Server URL `addRowFull()` row, spanning columns 1–3 (same layout as `helpText` in `addRowFull()`). Hidden by default.

**Source:** `BackendConfigPanel.kt` lines 75–83 and 232–234.

### 4. Bearer Token Show/Hide Toggle

Pattern: `JPasswordField` + `JButton("Show")` in a horizontal panel. Matches the API key field convention established by all `JPasswordField` fields in `BackendConfigPanel.kt` and `SettingsPanel.kt`.

```
JPanel(BoxLayout.X_AXIS) — tokenPanel
  JPasswordField(20)     — applyFieldStyle(); echoChar = '*'; Typography.mono font
  Box.createRigidArea(Dimension(Spacing.sm, 0))   // 8 px
  JButton("Show")        — secondaryButton(); 44×28 min size (touch target)
    actionListener:
      if echoChar == '*': field.echoChar = 0.toChar(); button.text = "Hide"
      else:               field.echoChar = '*';         button.text = "Show"
```

`JPasswordField` styling via `applyFieldStyle()` (overrides border, background, foreground, font). The field starts with `echoChar = '*'` (masked). No plaintext is ever shown in logs or saved to any non-encrypted store.

**Source:** `BackendConfigPanel.kt` lines 94, 102, 106 etc. (JPasswordField pattern); `SettingsPanel.kt` tokenPanel pattern for the horizontal composition.

Validation fires on "Save Server" click, not on keystroke (except SSRF which is real-time advisory).

## Connection Status in MCP Tools Tab

Per CONTEXT.md decision ("implementer's choice; prefer surfacing status in the existing MCP Tools tab if cheap"), the following minimal status surface is prescribed:

- The existing MCP Tools tab tool list shows external tools with the `ext:<server>:<tool>` prefix as tool name.
- No separate status column is added to the MCP Tools tab table for v1 (keep cheap).
- External tool rows in the MCP Tools tab display a transport badge `toolBadge("ext", BadgeStyle.FULL)` to visually distinguish them from built-in Burp tools.
- Connection errors for external servers are logged to the AI Logger panel (existing `AuditLogger` integration) with entry type "external_mcp_error".
- The server list table in the Settings panel (described above) is the primary status surface (Status column with Connected / Error / Retrying states).

**Source:** CONTEXT.md "Claude's Discretion" — connection-status surfacing.

## Trust/Safety Affordances: External Tool Visual Distinction

External tools (prefix `ext:<server>:<tool>`) are visually distinguished from built-in Burp tools in every surface they appear:

| Surface | Built-in tool display | External tool display |
|---------|----------------------|----------------------|
| MCP Tools tab tool list | Tool name; `toolBadge("store", NATIVE)` or `toolBadge("full", FULL)` as applicable | Tool name with `ext:` prefix; `toolBadge("ext", FULL)` badge in `Colors.onSurfaceVariant` |
| Agent tool preamble (text) | `tool_name` | `ext:<server>:<tool>` (existing text, no UI chrome needed) |
| AI Logger panel | Entry annotated "external_mcp_call" | Same log row style; no extra chrome |

The `toolBadge("ext", BadgeStyle.FULL)` renders with `Colors.cardSurface` background and `Colors.onSurfaceVariant` text at `Typography.caption` size — neutral, not alarming, but visually distinct from NATIVE (green) badges.

## Destructive Action Contract

**Remove server** is the only destructive action in this phase.

Confirmation approach: `JOptionPane.showConfirmDialog` (standard Swing modal):
- Title: "Remove Server"
- Message: "Remove '{displayName}'? The server will be disconnected and all its tools removed from the agent."
- Options: ["Remove", "Cancel"] — "Remove" maps to `JOptionPane.YES_OPTION`
- Message type: `JOptionPane.WARNING_MESSAGE`
- Default focus: "Cancel" (safe default)

No other destructive actions exist in this panel. Editing a server does not destroy it (Cancel restores previous state).
