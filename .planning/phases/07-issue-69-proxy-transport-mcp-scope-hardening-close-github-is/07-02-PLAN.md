<objective>
Close BUG-69-02: give users with 1278-token-class (small) local models a one-click way to shrink chat context (1500/750 chars vs 4000/8000), and let them configure tight MCP body caps below the current 1 MB minimum so MCP tool results fit those models too.

Purpose: GitHub issue #69 sub-concern 3 is "the context the AI sees is huge — 4 KB request + 8 KB response per item, blown out for small local models, plus the MCP body-cap spinner starts at 1 MB". Today users have to manually rebuild ContextCollector inputs or pre-truncate every prompt. A single Chat-section checkbox + a denominated-in-KB MCP cap solves both with a 5-line UI change and one new setting key.

Output:
- `AgentSettings.smallModelMode: Boolean = false` plus `KEY_CHAT_SMALL_MODEL_MODE = "chat.small.model.mode"` preference key; existing schema (v3) does not need a bump because the key absent → default false (defensive load).
- `SettingsPanel`: new "Small model mode" checkbox in the existing Backend/Chat section; `mcpMaxBodyMb` spinner replaced with a KB-denominated `mcpMaxBodyKb` (range 32..102400, default `(settings.mcpSettings.maxBodyBytes / 1024).coerceAtLeast(32)`).
- `AgentSettings.loadMcpSettings`: coerceIn lower bound lowered from `256 * 1024` to `32 * 1024`.
- `UiActions.contextOptionsFromSettings`: when `settings.smallModelMode` is true, override the two body-char caps to 1500 / 750.
- Two test classes: extended migration test + new SmallModelContextOptionsTest.

OUT OF SCOPE for this plan (deferred per 07-CONTEXT.md): per-backend max-context-window auto-detection. The setting is a manual toggle.
</objective>
