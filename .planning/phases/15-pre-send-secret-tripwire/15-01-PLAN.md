<objective>
Build the AWT-free detector core for the pre-send secret tripwire (PRIV-03): a pure-Kotlin Shannon-entropy helper (`redact/Entropy.kt`) and the thin orchestrator (`redact/SecretTripwire.kt`) that fuses `SecretShapes.findSurviving` (known shapes, Phase 13) with the entropy heuristic (unknown high-entropy tokens). This is the single detector that all three outbound paths (interactive dialog, passive scanner, MCP) reuse in Wave 2, so the interactive and non-interactive paths agree by construction (SC4).

This is a Wave-0-first plan: the test files are created first per 15-VALIDATION.md (`nyquist_compliant: true`), then the implementation is written to pass them.

Purpose: This IS the load-bearing security control of the phase. One pure detector keeps detection consistent across paths and headless-testable, and the no-leak discipline (category names + numeric score only, never the raw token) is enforced at the type level in `ScanResult` so no consumer can accidentally exfiltrate a secret into the audit log or UI.

Output: `redact/Entropy.kt`, `redact/SecretTripwire.kt`, `redact/EntropyTest.kt`, `redact/SecretTripwireTest.kt`. Zero new dependencies (entropy is `kotlin.math.ln`; shapes are the in-repo Phase 13 object).
</objective>
